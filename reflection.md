# Reflection

This project consisted of creating a housing affordability dashboard using Python, Pandas, Streamlit, and Plotly. The goal was to design an application that could help users compare housing affordability across different cities and ZIP codes.

One of the biggest challenges during development was debugging the ETL pipeline.  A issue I had was incorrect affordability formulas caused all housing entries to fall into only one category, which led to debugging of both the transformation logic and exported CSV outputs.


Another challenge I had was th old CSV data was still being displayed even after changes to the ETL script which led to me having to rebuild the dataset

Debugging techniques used throughout development were inspecting CSV files directly,printing DataFrame outputs and restarting Streamlit.


This project improved skills in ETL pipeline design, pandas data transformation, python debugging, streamlit dashboard development, interactive data visualization and project organization and repository structure

The final result is a scalable housing market intelligence dashboard that can be extended with real-world APIs and advanced analytics in future versions.
